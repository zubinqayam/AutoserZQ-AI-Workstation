ZQ Workstation v3.0

AI Multi-Browser Command Center & Research Operating System

Master Implementation Directive for Replit AI

---

Executive Vision

ZQ Workstation is not a web browser.

It is not a chatbot.

It is not a scraping tool.

It is an AI Multi-Browser Command Center powered by a Research Operating System.

The goal is to allow a user to control multiple live browser sessions simultaneously from one AI control room using natural language.

The browser layer must behave like Chrome, Edge, or Firefox while the AI orchestrates tasks across all browser panels in parallel.

The user must always feel they are watching and controlling real browsers—not screenshots or recreated pages.

---

Core Product Vision

Imagine an operations center where four independent browsers are always visible.

Each browser is fully interactive.

Each browser can:

- Browse any website
- Log into accounts
- Maintain independent cookies
- Maintain independent sessions
- Execute AI instructions
- Be manually controlled at any time

The AI acts as an operations manager.

Instead of opening websites manually, the user simply types commands into the Control Room.

Example:

«Search Google for Sohar Port expansion.»

The AI automatically decides:

Browser 1
→ Open Google
→ Search Sohar Port expansion

Browser 2
→ Open Bing
→ Search Sohar Port expansion

Browser 3
→ Search Perplexity

Browser 4
→ Search LinkedIn

All browsers execute simultaneously.

Expand / Focus Mode

This is a mandatory feature.

When a browser panel is expanded:

DO NOT

Open a new page.

DO NOT

Create another browser.

DO NOT

Reload the website.

Instead:

Resize the existing browser instance.

Example:

Browser 2

Currently:

- Logged into LinkedIn
- Uploading an image
- AI typing a comment

When Expand is clicked:

The exact same browser becomes fullscreen.

Everything continues.

The upload continues.

Typing continues.

Scrolling continues.

Network requests continue.

Nothing restarts.

When the user exits fullscreen:

The browser simply returns to the grid.

No refresh occurs.

---

Three Display Modes

Grid Mode

2×2 browsers.

All visible.

---

Focus Mode

One browser occupies approximately 80–90% of the screen.

The remaining browsers stay visible as live thumbnails.

They continue working.

---

Fullscreen Mode

One browser fills the workspace.

The remaining browsers continue running in the background.

Returning restores the previous layout instantly.

---

Final Implementation Directive

The user experience must always resemble a professional command center.

The browsers must remain live, interactive, and persistent.

Expanding a browser must resize the existing instance without creating a new page or reloading the session.

The AI Control Room should orchestrate all browser activity using natural language while the Research Operating System handles routing, evidence capture, analysis, and automation in the background.

The browser is the visible workspace.

The Research OS is the intelligence engine.

Both must function together as one seamless product.
